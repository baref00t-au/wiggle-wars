# Wiggle Wars — A brief for teachers

**What it is:** a fast, friendly classroom game where each player steers a growing
line that can only turn left or right — last one wiggling wins. It runs in any web
browser with **no install, no accounts, and no internet needed** to play.

**Why it's different:** the game doubles as a **digital-literacy tool**. It teaches
students to recognise the manipulative "engagement" tricks used in many apps and
games — and it deliberately contains **none of them itself**, so it's an honest
example to learn from.

---

## In 60 seconds

- Open the page → **Play** → pick 2–4 players (humans and/or bots) → go.
- Tap **📚 Learn** for the built-in lessons (no worksheets needed).
- It collects **nothing** about your students. Everything stays on the one device.

---

## The two ways to use it

### 1. Just play (5 min energiser / brain break)
Up to 4 on one device — keyboard (`A/D`, `←/→`, `J/L`, `Z/C`) or touch zones. Add
**🤖 bots** (Easy / Normal / Hard, or **Auto** which quietly matches the players'
level) so even one student has an opponent. First to the target score wins.

### 2. The "Learn" hub (the lesson, built into the game)
Five short, self-contained, no-score modes:

| Mode | What students do |
|---|---|
| 🔎 **Spot the Tricks** | Meet the 11 tricks. Each has a tiny *fake* demo they poke — it then names and explains the trick. A **Younger/Older** toggle shows 6 or all 11. |
| 🌍 **Trick Hunt** | Spot the same tricks out in social media, shopping, and streaming. |
| ✏️ **Design Lab** | Redesign a trick into an honest feature and save their "Honest Design" (on the device only). |
| 💚 **Fun vs. Can't-Stop** | Gentle self-awareness: telling *enjoying* a game from *not being able to stop*. No fear, no shame. |
| ⚙️ **How This Game Works** | A peek at the real tech — game loop, collisions, fairness — ending in a live "tweak a rule" sandbox. |

There's also a **🤝 Our Honest Promise** page: what the game chose *not* to build,
and why — the game practising what it teaches.

A full 45–60 minute lesson plan is in
[lesson-spot-the-trick.md](lesson-spot-the-trick.md), and the design ethics it's
built on are in [../ETHICS.md](../ETHICS.md).

### Running a class tournament
Just use **Play** on the projector. You control the pace: each round waits for a
tap or **Space** before the next begins (nothing auto-starts), and **P** pauses and
resumes at any time. Set a higher target score for a longer tournament.

## Privacy (the short version for admin/parents)
- **No accounts, no logins, no sign-up.**
- **No analytics, no trackers, no third-party anything, no network calls** during
  play.
- The only things stored are on **this device** (sound/colour/difficulty settings
  and any "Honest Design" a student saves) — never transmitted. Students' nicknames
  are **never** saved.
- There's a visible **"Clear everything on this device"** button (in Learn).
- This is designed to sit clear of children's-privacy rules (COPPA, GDPR-K, the AU
  Privacy Act) by collecting nothing.

## Accessibility
- Colour-blind-safe palette **plus a shape** for each player (so colour is never the
  only cue), shown on the line, the scoreboard, and the colour picker.
- A **reduced-motion** toggle (and it respects the device's reduced-motion setting).
- Keyboard-navigable menus, large touch targets, high contrast on the dark theme.

## Running it
- **In class:** open the deployed link (any web host), or run locally. Best
  full-screen on a laptop/Chromebook or tablet. (It goes fullscreen on Start; press
  **Esc** to leave.)
- **No internet?** Fine — once the page has loaded, it plays fully offline.
- **Devices:** works on locked-down Chromebooks, managed iPads, and laptops.

### Install it like an app (Android & iPad)
Wiggle Wars is an installable web app — no app store, no account needed:
- **iPad / iPhone (Safari):** open the link → **Share** → **Add to Home Screen**.
- **Android / Chromebook (Chrome):** open the link → menu (⋮) → **Install app** /
  **Add to Home screen**.

It then launches full-screen from the home-screen icon and **works completely
offline** afterwards.

### Play on separate devices (WiFi or internet)
Students can play on their own devices — on the same WiFi, or over the internet with
a code (e.g. a club running across homes):
1. One device taps **📡 WiFi / Online → Host a game** and reads out the short **code**.
2. The others tap **WiFi / Online → Join a game**, type their name and the code.
3. The host taps **Start** — everyone plays together; the host paces the rounds.

The very first connection uses the internet (a quick handshake to link the devices);
the game itself is **device-to-device**, and **no student data is sent or stored**
(see [`../ETHICS.md`](../ETHICS.md)). It's for people sharing a code — there's no
public lobby of strangers. On WiFi with **no internet**, use same-device play. For
the technical details (and the rare case where a network needs a relay), see
[`online-play.md`](online-play.md).

## A note on tone
The aim isn't to scare students off games or to moralise — it's to hand them the
vocabulary and the agency. A child who can say *"that's just a streak trying to
guilt me"* has most of the defence already. Naming a manipulation is what disarms it.
